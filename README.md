# ✨ 프로젝트 소개

기존 브랜드 사이트의 고급스러운 디자인 감성을 유지하면서, 단순한 UI 구현을 넘어 **실제 쇼핑몰의 구매 흐름**을 경험할 수 있도록 리뉴얼한 프로젝트입니다.

# 📖 프로젝트 개요

**Bowers & Wilkins All-in-One Renewal**은 프리미엄 오디오 브랜드 **Bowers & Wilkins**를 모티브로 제작한 React 기반 커머스 프로젝트입니다.

단순한 UI 클론이 아닌 **상품 탐색 → 장바구니 → 결제 → 주문 완료**까지 이어지는 실제 쇼핑몰 구매 흐름을 구현했으며, Firebase를 활용한 회원 인증 및 게시판, Polar.sh 결제 시스템, Context API 기반 전역 상태 관리 등 실제 서비스에서 사용하는 기능을 직접 구현했습니다.

또한 React 컴포넌트 구조와 SPA 라우팅을 적용하여 페이지 이동 없이 자연스러운 사용자 경험을 제공하고, Three.js와 GSAP를 활용한 인터랙션으로 브랜드의 프리미엄 감성을 표현했습니다.

# 🔧 Trouble Shooting

## 1. Windows 환경에서만 빌드 실패

### 문제

프로젝트는 정상적으로 실행되었지만 Windows 환경에서만 빌드가 실패하는 문제가 발생했습니다.

### 해결

파일명 규칙을 수정하고 플랫폼별 파일 처리 방식을 고려하여 구조를 변경했습니다.

### 결과

운영체제와 관계없이 동일하게 빌드가 가능하도록 개선했습니다.

---

## 2. Polar.sh Secret Key 노출 문제

### 문제

브라우저에서 결제 API를 직접 호출하면 Secret Key가 노출되는 문제가 발생했습니다.

### 해결

Secret Key는 `.env`와 서버에서만 관리하고, 브라우저에서는 서버를 통해 결제를 요청하도록 구조를 변경했습니다.

### 결과

민감한 정보를 안전하게 보호하면서 실제 결제 기능을 구현할 수 있었습니다.

---

## 3. GitHub Pages에서 결제가 동작하지 않는 문제

### 문제

GitHub Pages는 정적 호스팅만 지원하기 때문에 서버 기능이 필요한 결제가 정상적으로 동작하지 않았습니다.

### 해결

결제 기능이 포함된 서비스는 **Vercel**에 배포하고, GitHub Pages는 프로젝트 소개 및 데모 용도로 분리하여 운영했습니다.

### 결과

배포 환경에 맞는 구조를 적용하여 기능과 포트폴리오를 모두 유지할 수 있었습니다.

---

## 4. 다크모드에서 이미지 색상까지 반전되는 문제

### 문제

CSS Filter를 이용해 다크모드를 구현하자 상품 이미지와 영상까지 함께 반전되었습니다.

### 해결

이미지와 영상 요소에 추가적인 반전 처리를 적용하여 원래 색상을 유지하도록 수정했습니다.

### 결과

콘텐츠는 다크모드로 변경되면서도 상품 이미지는 정상적인 색상을 유지할 수 있었습니다.

---

## 5. Firebase 설정 누락 시 앱 전체 오류

### 문제

Firebase 설정이 준비되지 않은 상태에서 앱이 실행되면 로그인 기능에서 오류가 발생하며 전체 애플리케이션이 중단되었습니다.

### 해결

Firebase 초기화 여부를 먼저 확인하는 안전장치를 추가하여 설정이 없는 경우에도 기본 화면은 정상적으로 동작하도록 구현했습니다.

### 결과

외부 서비스 오류가 전체 서비스에 영향을 주지 않는 구조를 만들 수 있었습니다.

---

## 6. 이미지 변경 후 화면이 갱신되지 않는 문제

### 문제

이미지를 교체해도 개발 서버에서 이전 이미지가 계속 표시되는 문제가 발생했습니다.

### 해결

이미지 변경 시 개발 서버를 재시작하는 규칙을 적용하고 이미지 관리 방식을 정리했습니다.

### 결과

이미지 변경 사항이 안정적으로 반영되도록 개선했습니다.

# ▶️ 실행 방법

## 1. 저장소 클론

```bash
git clone https://github.com/hong4745-cyber/onepage.git
```

---

## 2. 프로젝트 폴더 이동

```bash
cd onepage
```

---

## 3. 패키지 설치

```bash
npm install
```

---

## 4. 개발 서버 실행

```bash
npm run dev
```

---

## 5. 환경 변수 설정

Polar.sh와 Firebase를 사용하려면 `.env` 파일에 자신의 설정 정보를 입력해야 합니다.

```env
VITE_FIREBASE_API_KEY=your_key
VITE_FIREBASE_AUTH_DOMAIN=your_domain

VITE_POLAR_ACCESS_TOKEN=your_token
```

> **참고**
>
> 보안을 위해 실제 API Key와 Secret Key는 Repository에 포함하지 않았습니다.
